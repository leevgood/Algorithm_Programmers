# 문자 개수 세기 (Count Characters)

## 1. 문제 설명

알파벳 대소문자로만 이루어진 문자열 `my_string`이 주어집니다.

문자열에 포함된 알파벳의 개수를 다음 순서로 저장한 길이 `52`의 정수 배열을 반환해야 합니다.

```text
A, B, C, ..., Z, a, b, c, ..., z
```

배열의 인덱스는 다음과 같이 구성됩니다.

| 인덱스 범위    | 저장되는 문자     |
| --------- | ----------- |
| `0 ~ 25`  | 대문자 `A ~ Z` |
| `26 ~ 51` | 소문자 `a ~ z` |

---

## 2. 제한사항

* `1 ≤ my_string의 길이 ≤ 1,000`
* `my_string`은 알파벳 대문자와 소문자로만 구성됩니다.

---

## 3. 핵심 아이디어

영문 알파벳은 문자 코드상 연속적으로 배치되어 있습니다.

```text
'A' → 65
'B' → 66
...
'Z' → 90

'a' → 97
'b' → 98
...
'z' → 122
```

따라서 문자의 인덱스를 다음과 같이 계산할 수 있습니다.

### 대문자

```text
문자 - 'A'
```

예를 들어 `P`의 위치는 다음과 같습니다.

```text
'P' - 'A' = 15
```

### 소문자

소문자는 배열의 `26`번 인덱스부터 시작하므로 다음과 같이 계산합니다.

```text
문자 - 'a' + 26
```

예를 들어 `a`의 위치는 다음과 같습니다.

```text
'a' - 'a' + 26 = 26
```

---

# 4. C# 풀이

## 소스 코드

```csharp
using System;

public class Solution
{
    public int[] solution(string my_string)
    {
        // 대문자 26개와 소문자 26개의 개수를 저장할 배열
        int[] answer = new int[52];

        foreach (char c in my_string)
        {
            if (c >= 'A' && c <= 'Z')
            {
                // 대문자 A~Z는 인덱스 0~25에 저장
                answer[c - 'A']++;
            }
            else
            {
                // 소문자 a~z는 인덱스 26~51에 저장
                answer[c - 'a' + 26]++;
            }
        }

        return answer;
    }
}
```

---

## 주요 문법 설명

| 문법                     | 설명                                        |
| ---------------------- | ----------------------------------------- |
| `new int[52]`          | 길이가 52인 정수 배열을 생성합니다. 모든 값은 `0`으로 초기화됩니다. |
| `foreach`              | 문자열에 포함된 문자를 하나씩 순회합니다.                   |
| `char`                 | 하나의 문자를 저장하는 자료형입니다.                      |
| `c >= 'A' && c <= 'Z'` | 문자가 대문자인지 확인합니다.                          |
| `c - 'A'`              | 대문자의 배열 인덱스를 계산합니다.                       |
| `c - 'a' + 26`         | 소문자의 배열 인덱스를 계산합니다.                       |
| `answer[index]++`      | 해당 문자의 개수를 1 증가시킵니다.                      |

---

## C# 코드 동작 과정

입력 문자열이 다음과 같다고 가정합니다.

```text
"Programmers"
```

첫 번째 문자 `P`는 대문자입니다.

```csharp
'P' - 'A' = 15
```

따라서 다음 위치의 값이 증가합니다.

```csharp
answer[15]++;
```

다음 문자 `r`은 소문자입니다.

```csharp
'r' - 'a' + 26
= 17 + 26
= 43
```

따라서 다음 위치의 값이 증가합니다.

```csharp
answer[43]++;
```

같은 방식으로 문자열의 모든 문자를 확인합니다.

# 5. Python 풀이

## 소스 코드

```python
def solution(my_string):
    # 대문자 26개와 소문자 26개의 개수를 저장할 리스트
    answer = [0] * 52

    for char in my_string:
        if 'A' <= char <= 'Z':
            # 대문자 A~Z는 인덱스 0~25에 저장
            index = ord(char) - ord('A')
        else:
            # 소문자 a~z는 인덱스 26~51에 저장
            index = ord(char) - ord('a') + 26

        answer[index] += 1

    return answer
```

---

## 주요 문법 설명

| 문법                          | 설명                            |
| --------------------------- | ----------------------------- |
| `[0] * 52`                  | 값이 모두 `0`인 길이 52의 리스트를 생성합니다. |
| `for char in my_string`     | 문자열의 문자를 하나씩 순회합니다.           |
| `'A' <= char <= 'Z'`        | 문자가 대문자인지 확인합니다.              |
| `ord(char)`                 | 문자를 유니코드 정수 값으로 변환합니다.        |
| `ord(char) - ord('A')`      | 대문자의 배열 인덱스를 계산합니다.           |
| `ord(char) - ord('a') + 26` | 소문자의 배열 인덱스를 계산합니다.           |
| `answer[index] += 1`        | 해당 문자의 개수를 1 증가시킵니다.          |

---

## Python 코드 동작 과정

문자 `P`의 인덱스는 다음과 같이 계산됩니다.

```python
ord('P') - ord('A')
```

계산 결과는 `15`이므로 다음 값이 증가합니다.

```python
answer[15] += 1
```

문자 `r`의 인덱스는 다음과 같이 계산됩니다.

```python
ord('r') - ord('a') + 26
```

계산 결과는 `43`이므로 다음 값이 증가합니다.

```python
answer[43] += 1
```

# 7. 복잡도 분석

문자열의 길이를 `N`이라고 할 때 모든 문자를 한 번씩 확인합니다.

* **시간 복잡도:** `O(N)`
* **공간 복잡도:** `O(1)`

결과 배열의 길이는 문자열 길이와 관계없이 항상 `52`이므로 상수 공간을 사용합니다.

# 8. 참고 사항

* 대문자와 소문자는 서로 다른 문자로 처리해야 합니다.
* `A`는 배열의 `0`번 인덱스에 저장됩니다.
* `Z`는 배열의 `25`번 인덱스에 저장됩니다.
* `a`는 배열의 `26`번 인덱스에 저장됩니다.
* `z`는 배열의 `51`번 인덱스에 저장됩니다.
* 입력값은 알파벳으로만 이루어져 있으므로 숫자나 특수문자를 별도로 처리할 필요가 없습니다.
